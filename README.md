# 더기 (Deogi)

앱 이름은 **duck + "이" → 더기**처럼 부르기 쉽게 지었어요. (영문 슬러그: `deogi`) **해오리** 닉네임과 오리 마스코트랑도 잘 어울립니다.

중국어 ↔ 한국어 통역용 React Native(Expo) 앱입니다. Google Gemini로 번역하고, 기기 TTS로 발음을 들을 수 있습니다.

## 현재 구현

- **텍스트 번역** · **음성 녹음 후 멀티모달 번역** (짧은 녹음 → Gemini 오디오 입력)
- **방향**: 中→한 / 한→中(간체)
- **히스토리**: AsyncStorage
- **발음**: `expo-speech` (시스템 TTS, `zh-CN` / `ko-KR` 등)
- **UI**: 오리 🦆 마스코트 애니메이션, 아이콘 중심 레이아웃 (`@expo/vector-icons`)

환경 변수는 `.env.example` 참고. `EXPO_PUBLIC_GEMINI_API_KEY` 필요.

```bash
pnpm install
pnpm start
```

## 향후 방향 (로드맵)

**통역사 옆 자막에 가깝게** 들리는 말을 거의 실시간으로 옮기고 싶다는 목표가 있습니다. 그래서 아래 쪽으로 확장하는 것을 염두에 둡니다.

1. **스트리밍 STT (음성 → 텍스트)**  
   - 마이크 입력을 끊지 않고 **연속으로 문자열로 뽑는** 쪽 (예: 온디바이스/클라우드 Speech-to-Text).  
   - 나온 중국어(또는 한국어) 텍스트를 짧은 단위로 Gemini **텍스트** 번역에 넣으면, 청크 녹음 반복보다 자연스러운 “자막”에 가깝게 만들 수 있음.

2. **Gemini Live / 스트리밍 오디오 API**  
   - **양방향·저지연** 오디오 파이프라인이 필요할 때 후보.  
   - WebSocket 등으로 오디오 스트림을 주고받는 모델·엔드포인트와, Expo/React Native에서의 **오디오 캡처·재생·세션 관리**가 추가 과제.

3. **실무적으로 필요한 것들**  
   - API 쿼터·비용, 백그라운드 녹음 정책, iOS/Android 권한·오디오 세션.  
   - Expo Go 한계를 넘으려면 **development build** + 네이티브 모듈 조합을 검토할 수 있음.

> 지금 앱은 “녹음 한 덩어리 → 한 번 번역”에 가깝고, 위 방향은 **별도 설계·프로토타입** 단계에서 진행하는 것을 권장합니다.
